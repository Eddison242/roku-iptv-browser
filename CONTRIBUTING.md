# Contribution Guidelines for Roku IPTV Browser

Thank you for considering contributing to Roku IPTV Browser! We welcome contributions from everyone!

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please be respectful of everyone. We want this project to foster an inclusive environment.

## Ways to Contribute

There are several ways you can contribute:

1. **Reporting Bugs:** If you notice a bug, please create a detailed issue to help us understand and fix the problem.
2. **Feature Requests:** If you have a suggestion for a new feature, please let us know!
3. **Code Contributions:** We welcome pull requests!
4. **Documentation Improvements:** Help us improve the documentation for better user understanding.

## Development Guidelines

- **Fork the Repository:** Create a personal copy of the repository.
- **Create a Branch:** Always work on a separate branch for your changes. Use clear naming for branches (e.g., `feature/add-search-functionality`).
- **Follow Coding Standards:** Ensure code follows the project's style guidelines.

## Testing Procedures

- **Unit Tests:** Add unit tests for any new code or features.
- **Run Tests:** Ensure all existing tests pass before submitting your pull request.
- **Test Environment:** Always test in a local environment before pushing changes.

## Submission Process

1. **Create a Pull Request:** Once you're ready to submit, create a pull request from your feature branch to the `main` branch.
2. **Provide a Clear Description:** Mention the purpose of the pull request and reference any related issues.
3. **Adhere to Review Feedback:** Be open to making requested changes as part of the review process.

Thank you for contributing to Roku IPTV Browser! Your help is invaluable!
